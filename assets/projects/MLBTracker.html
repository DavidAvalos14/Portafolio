MLB Tracker

Baseball Statistics and Live Scores - Android App with Kotlin

2024 Kotlin, Android Studio, REST APIs Repository

Overview

MLB Tracker is a comprehensive Android application designed for baseball enthusiasts to follow Major League Baseball games in real-time. Developed using Kotlin in Android Studio, this app integrates with MLB's official API to provide live scores, detailed player statistics, team standings, and game schedules. The project was born from a passion for baseball and the desire to create a streamlined mobile experience for tracking favorite teams and players.

The application showcases modern Android development practices including MVVM architecture, Retrofit for API calls, Coroutines for asynchronous operations, and Room database for local data persistence. Users can customize their experience by following specific teams, setting up game notifications, and accessing historical statistics and game highlights.

Features

  • Live Score Updates: Real-time scores and play-by-play updates for ongoing MLB games
  • Team Standings: Current league standings with win-loss records and playoff positions
  • Player Statistics: Comprehensive batting and pitching stats for all MLB players
  • Game Schedule: Complete season schedule with past results and upcoming matchups
  • Team Profiles: Detailed information about all 30 MLB teams including rosters
  • Favorite Teams: Personalize the app by following your favorite teams
  • Push Notifications: Receive alerts for game starts, score updates, and final results
  • Game Highlights: Video highlights and key plays from recent games
  • Player Search: Quick search functionality to find specific players and their stats
  • Historical Data: Access to past seasons' statistics and records
  • Offline Mode: View cached data when internet connection is unavailable
  • Dark Mode: Eye-friendly dark theme option for nighttime use
  • Box Scores: Detailed inning-by-inning breakdowns of each game

Technologies Used

Kotlin Android Studio MVVM Architecture Retrofit Coroutines LiveData ViewModel Room Database Glide RecyclerView Material Design MLB API Firebase Cloud Messaging Gson

Technical Implementation

  • API Integration: Implemented Retrofit for making HTTP requests to MLB's REST API with proper error handling and response parsing
  • Asynchronous Operations: Used Kotlin Coroutines for non-blocking API calls and database operations, ensuring smooth UI performance
  • Local Storage: Implemented Room database for caching data and enabling offline functionality with proper entity relationships
  • Repository Pattern: Created a clean data layer that abstracts data sources and provides a single source of truth
  • Push Notifications: Integrated Firebase Cloud Messaging for real-time notifications about game events
  • Image Loading: Used Glide library for efficient loading and caching of team logos and player photos

Challenges Overcome

  • API Rate Limiting: Implemented intelligent caching strategies to minimize API calls while maintaining data freshness
  • Real-Time Updates: Developed an efficient polling mechanism for live score updates without draining battery
  • Complex Data Structures: Parsed and organized hierarchical JSON responses from the API into usable data models
  • State Management: Maintained consistent UI state across configuration changes and app lifecycle events
  • Performance Optimization: Optimized RecyclerView with ViewHolder pattern and DiffUtil for smooth scrolling
